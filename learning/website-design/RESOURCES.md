# Website & UI Design Resources

## Knowledge

- [taste-skill by Leonxlnx](https://github.com/Leonxlnx/taste-skill)
  The "anti-slop" skill pack — installed in this branch at `.claude/skills/`. Use for: every implementation session (`design-taste-frontend`, `image-to-code`) and for writing mockup prompts (`imagegen-frontend-web`, `imagegen-frontend-mobile`). Its `research/` folder has background writing on design principles.
- [Refactoring UI (book) — Wathan & Schoger](https://www.refactoringui.com/)
  The single most recommended design book for non-designers, by the Tailwind CSS creators. Use for: the core taste curriculum — hierarchy, spacing, color, typography. Paid, but the [free preview chapters and their blog posts](https://www.refactoringui.com/) cover a lot.
- [Laws of UX](https://lawsofux.com/)
  Psychology-backed design principles (Hick's Law, Fitts's Law, Jakob's Law) each on one beautiful page. Use for: understanding *why* layouts work, one law at a time.
- [Nielsen Norman Group articles](https://www.nngroup.com/articles/)
  Gold-standard research-backed UX writing. Use for: settling "is this actually good UX?" questions.
- [Untitled UI — 21 books for UI designers](https://www.untitledui.com/blog/ui-design-books)
  Curated reading list. Use for: choosing deeper reading after the fundamentals land.
- [MDN — HTML & CSS first steps](https://developer.mozilla.org/en-US/docs/Learn_web_development)
  The reference for reading the code Claude writes. Use for: minimum viable HTML/CSS literacy (lesson on reading code, not writing it).

## Inspiration (trains the eye)

- [Awwwards](https://www.awwwards.com/) — award-winning sites; study what premium looks like.
- [Land-book](https://land-book.com/) — large gallery of landing pages, filterable by style.

## Wisdom (Communities)

- [r/design_critiques](https://reddit.com/r/design_critiques)
  Post work, get specific feedback. Use for: critique on mockups and shipped pages.
- [r/web_design](https://reddit.com/r/web_design)
  Broader discussion. Use for: workflow questions, seeing others' image-to-code experiments.
